# Help outline a new post (as markdown): $ARGUMENTS

1. Ask questions about the provided ideas until you have clarity:
   - What's the core insight or argument?
   - What post type fits best (see blog-writing-reference.md for types)?
   - Is there a personal anecdote or experience to anchor it?
   - Who is the intended reader?
2. Follow the blog writing rules in `rules/blog-writing-rules.md`
3. Produce a structured outline with: headline options, opening hook, main sections, closing move
4. Flag any sections that need author-supplied personal content with `[PERSONAL ANECDOTE: ...]` placeholders
5. Present the outline for review before proceeding
